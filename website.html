How to upload HTML code to a website (GitHub Pages):


Creating a repository on github:

Go to github.com and log on. (create an account if you don't have one yet)

Next click on your icon in the upper right corner. To open the drop down menu.

Select Your repositories. Repositories are like folders where files and documents can be stored.

Click on new for new repository (has a little book with a bookmark icon next to it).

Add a repository name that you can use to identify the repository with later.

Make sure the repository is public.

Leave everything else as is and click on Create repository.


Uploading code to a GitHub repository

In the Quick setup section of the new repository choose the link for “upload an existing file” (You can also type /upload at the top of the repository web address)

Now simply drag and drop files to the “Drag files here to add them to your repository” section or click on the “choose your files” hyperlink text to search for them on your computer.

Don't drag the folder with the files of code into the “choose your files” section. Instead, only drag the files contianing the actual code into the “choose your files” section.

Once files are added the “choose your files” will turn into the “Drag additional files here to add them to your repository.” section.

The files should now show up below the “Drag additional files here to add them to your repository.” section.

Click on “Commit changes” once you have the needed files uploaded.


How to add the uploaded code to GitHub Pages:

From your repository's page click on Settings button in the top right.

From Settings under the Code and automation section on the left choose Pages.

In GitHub Pages under the Source section in the middle click on the drop down that starts with “None” selected.

In the drop-down choose “main” (It may be called something else like master but it should be the option that isn't “none”)

Next to the drop down that now has Branch:main (unless main wasn't the name) leave the “/(root)” drop down as the default then click Save.

The code will then publish to the internet. It make take a few minutes.

After a few minutes you can refresh the page and it should say “Your site is published at” followed by a hyperlink. Now anyone with that hyperlink on the internet can view our website.

If you don not see the hyperlink you can also access it using your github_username.github.io/repository_name

You can right click on the link in the GitHub Pages and open in a new tab to open the website in a new tab.

When you open it likely it will have a 404 File not found error at this point.

In the repository files you will have the names of your files. If you add a slash (/) then the name of one of the files that are part of your html website and press enter it will take you to your website.

The GitHub repository works the same way as the computer in that the files on GitHub in the repository link up using the file names found in the repository just like they do in Visual Studio Code. This is the same for loading images which should be uploaded to the GitHub Repository and named as they are in Visual Studio Code.

Steps to put the webstie on internet:

  1. Write HTML & CSS code
  2. Upload code to GitHub

The 404 error mentioned before is because by default the website will look for a file called index.html. If this file is not uploaded to GitHub then it will return the error and you will have to manually add the file name. So to fix the problem we need to add an index.html file into our repository. You can rename a file to index.html if you want that file to automatically open by default. It may take a few momenets after updating the name before the changes are finished loading. Once they have been updated if you refresh your website's page it will load without the 404 error and instead using the html code in the index.html file.

Important information about this process:

A web server is just a piece of softwared you run on your computer. GitHub is a Web Server but run on their computers. Live Server is also a web server.

A web server can communicate with a browser. This is done by typing in a url.

The browser can ask the web server for files.

If you happen to get back an html file it will display the website instead of giving back the text of the file.

Web Servers need a computer for it to run. The computer running the web server is called a host.

Web Hosting Company provides you with computers to run the web servers and the server software.

The ip address is given anytime the internet is accessed.

The IP address like a phone number is able to be contacted by other devices connected to the internet. IP addresses are unique.

There are two kinds of IP addresses:

  1. IPv4 (4 has groups of typically 3 numbers seperated by decimals)
  2. IPv6 (has around 15 digits mostly numbers some letters seperated by colons)

Version 6 exist because ip addresses for version 4 are running out.

We need to link our domain name to GitHub's ip address.

We can set up a link between a domain name and an ip address in a Domain Registrar (If you bought your domain name from namecheap it will be the one you purchased through them).

The link between the Domain Registrar and the GitHub ip address is called a DNS record (Domain Name System Record).


How to Set up a DNS Record inside the Domain Registrar:

Login to your domain registrar (namecheap if you bought your name there.)

The remaining instructions for the domain registrar are using namecheap.

Once logged in go to the Settings for the domain name.

To go to Settings for the domain name in namecheap click on Domain List.

Scroll down to the domain name you want to use.

Click the Manage button to the right of the domain name.

Click on the Advanced DNS tab.

Scroll down to the table with headers of “Type”, “Host”, “Value”, and “TTL”.

You can also look on Google for Manage dns records YourDomainRegistrarName

Under the table there will be a “ADD NEW RECORD” button. Click this button.

Fill in the values for the DNS record.

The first value you will fill in is the Record type.

For an ip address use an “A Record.”

Next value will be the TTL value.

TTL stands for TIme To Live. This refers to how long after chaning the DNS records before they change accross the internet. short TTL means the DNS changes will be updated very quickly. A long TTL means it will be updated more slowly. Long TTLs are a little more efficient but for this project you can select 5 minutes. If it asks for seconds instead of minutes do 300 seconds which is 5 minutes.

Next the sub domain (Under the Host heading) is the part of the url in front of a domain name.

We want the sub domain to be blank. To do this using namecheap we use the at (@) symbol.

For other domain registrars you can just leave it blank.

The last value needed is the IP Address (of GitHub Pages).

To find the IP Address of GitHub Pages go to a new tab and search for dns lookup.

The website you want will under https://mxtookbox.com

DNS lookup tool (IPv4) DNS lookup tool (IPv6)

At this address you can search for a domain Name and see which IP addresses are currently linked to it.

Type in the GitHub user name followed by .github.io in the Domain Name search bar.

UserNameWithGitHub.github.io (This is the default name GitHub gives you)

Click on DNS Lookup.

It will show you the IP addresses associated with GitHub Pages.

For each IP address you will need to create a DNS record.

First click on one of the IP address results in the DNS lookup tool.

Then copy the IP address that appears on the left (the information after the ptr:)

Paste the Copied IP address from the DNS lookup tool Into the IP Adress under the Value header of the namecheap page.

If all four values are filled out click the green checkmark to Save Changes.

Repeat the steps for filling in the values for the reamining IP addresses that appeared in the DNS lookup tool.


Setting up the www sub-domain

The final step is to set up a sub-domain called www

Although setting up the www is not required it is recommended by GetHub Pages.

In namecheap you will need to click on ADD NEW RECORD again in the settings of your Domain Name.

Instead of an A Record like the other ones you set up, This record will be a CNAME Record.

Record types Info:

  1. A Record: Links a website to an IP address
  2. CNAME Record: Links a domain name to another domain name

The CNAME makes it easier to link to GitHub Pages without having to know the specific ip addresses.

In the sub-domain section (under the Host header) you will put www

Under the Value header section (IP address section) you will put your UserNameWithGitHub.github.io (This is the default name GitHub gives you).

In the TTL section you will choose 5 minutes (or 300 seconds).

Click the checkmark to save the changes.

You can only use CNAME Records for sub-domains

You should setup IP addresses for both IPv4 and IPv6. IPv6 will start being used more in the future.

Setting up HTTPS

Even though the site is live it is not secure.

To secure it you need to set up HTTPS.

HTTPS encripts all data flowing between the browser and the website.

To set up HTTPS go back to GitHub Pages.

In the repository settings of your website scroll down until you find the Enfoce HTTPS check box.

Check the box.

There is a behind the scenes process to set up HTTPS and it may take up to 15 minutes.

Once the process is finished just refresh the page and the checkbox should be enabled (You still have to check it).

Check the box and and it will say the DNS check is in progress.

Now go back to your website url and type https:// in front of it.

A lock icon will appear in front of the website indicating that the https is enabled.
