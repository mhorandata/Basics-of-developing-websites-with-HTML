How to upload HTML code to a website (GitHub Pages):


Creating a repository on github:

Go to github.com and log on. (create an account if you don't have one yet)

Next click on your icon in the upper right corner. To open the drop down menu.

Select Your repositories. Repositories are like folders where files and documents can be stored.

Click on new for new repository (has a little book with a bookmark icon next to it).

Add a repository name that you can use to identify the repository with later.

Make sure the repository is public.

Leave everything else as is and click on Create repository.


Uploading code

In the Quick setup section of the new repository choose the link for “upload an existing file” (You can also type /upload at the top of the repository web address)

Now simply drag and drop files to the “Drag files here to add them to your repository” section or click on the “choose your files” hyperlink text to search for them on your computer.

Don't drag the folder with the files of code into the “choose your files” section. Instead, only drag the files contianing the actual code into the “choose your files” section.

Once files are added the “choose your files” will turn into the “Drag additional files here to add them to your repository.” section.

The files should now show up below the “Drag additional files here to add them to your repository.” section.

Click on “Commit changes” once you have the needed files uploaded.


How to add the uploaded code to GitHub Pages:

From your repository's page click on Settings button in the top right.

From Settings under the Code and automation section on the left choose Pages.

In GitHub Pages under the Source section in the middle click on the drop down that starts with “None” selected.

In the drop-down choose “main” (It may be called something else like master but it should be the option that isn't “none”)

Next to the drop down that now has Branch:main (unless main wasn't the name) leave the “/(root)” drop down as the default then click Save.

The code will then publish to the internet. It make take a few minutes.

After a few minutes you can refresh the page and it should say “Your site is published at” followed by a hyperlink. Now anyone with that hyperlink on the internet can view our website.

If you don not see the hyperlink you can also access it using your github_username.github.io/repository_name

You can right click on the link in the GitHub Pages and open in a new tab to open the website in a new tab.

When you open it likely it will have a 404 File not found error at this point.

In the repository files you will have the names of your files. If you add a slash (/) then the name of one of the files that are part of your html website and press enter it will take you to your website.

The GitHub repository works the same way as the computer in that the files on GitHub in the repository link up using the file names found in the repository just like they do in Visual Studio Code. This is the same for loading images which should be uploaded to the GitHub Repository and named as they are in Visual Studio Code.

Steps to put the webstie on internet:

  1. Write HTML & CSS code
  2. Upload code to GitHub

The 404 error mentioned before is because by default the website will look for a file called index.html. If this file is not uploaded to GitHub then it will return the error and you will have to manually add the file name. So to fix the problem we need to add an index.html file into our repository. You can rename a file to index.html if you want that file to automatically open by default. It may take a few momenets after updating the name before the changes are finished loading. Once they have been updated if you refresh your website's page it will load without the 404 error and instead using the html code in the index.html file.
